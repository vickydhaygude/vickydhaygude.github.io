01 — Enterprise AI · Capgemini · 2025
GxP AI Compliance System
LLM Workflow Document AI Human-in-Loop
Problem

In pharma and life sciences, every document change must be classified as GxP-relevant or not. This was done manually — slow, inconsistent, and a constant compliance risk across large document sets.

Approach

Built an end-to-end LLM workflow that reads document changes, classifies them using a language model, auto-generates audit logs and URS documents, then routes everything through a human approval step before finalising.

Tech Stack
LangChainLLMsPython Document ProcessingHuman-in-Loop
System Pipeline
Document Input→ LLM Classifier→ GxP / Non-GxP→ Auto-generate Docs→ Human Approval→ Final Audit Log
✅

Reduced manual classification and documentation effort per compliance cycle. Full audit trail built into the workflow. Deployed in a regulated enterprise environment.

02 — Document Intelligence · Capgemini · 2025
AI Document Structuring Tool
Hybrid Pipeline PDF / DOCX Parsing LLM Restructuring
Problem

Enterprise document libraries contain thousands of PDFs and DOCX files with inconsistent layouts. This makes downstream search, indexing, or AI processing nearly impossible without significant manual cleanup.

Approach

Designed a two-stage hybrid pipeline. Stage 1 uses rule-based preprocessing to extract document structure — headings, tables, sections. Stage 2 passes extracted content to an LLM which restructures it into a clean, consistent output format ready for downstream use.

Tech Stack
PythonLLMs PDF ParsingDOCX ProcessingRule-based NLP
System Pipeline
PDF / DOCX Input→ Rule-based Extraction→ Raw Structure→ LLM Restructure→ Clean Output
✅

Eliminated hours of manual document cleanup per week. Previously unprocessable document libraries became available for AI workflows and search indexing.

03 — GenAI Application · Capgemini · 2025
RAG Document Q&A Chatbot
RAG LangGraph Streamlit
Problem

Teams working with large document sets waste significant time manually searching for answers. Keyword search misses context-dependent answers and doesn't support follow-up questions.

Approach

Built a full RAG system. Documents are chunked, embedded, and stored for semantic search. LangGraph orchestrates the multi-step retrieval and generation workflow. The LLM generates grounded answers from retrieved content only — no hallucination from general knowledge.

Tech Stack
LangGraphLangChain RAGVector Store StreamlitPython
System Pipeline
User Query→ Embed Query→ Vector Search→ Retrieve Chunks→ LLM Generate→ Grounded Answer
✅

Context-aware, multi-turn Q&A on large document sets. Answers are grounded in retrieved content. Delivered via a clean Streamlit UI.
